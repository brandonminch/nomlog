# Monorepo (pnpm + Turborepo) implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Nomlog into a single **pnpm + Turborepo** workspace with a unified **`docs/`** tree, root scripts for `lint` / `typecheck` / `build`, and deployment docs updated so **development** workflows use the monorepo root (production hosting commands change only where this plan says so).

**Architecture:** One root `package.json` + `pnpm-workspace.yaml` lists `nomlog-app`, `nomlog-api`, `nomlog-web`, `docs`, and `packages/*`. `turbo.json` orchestrates tasks across packages. Product docs from `nomlog-docs/` and workflow docs from `docs/superpowers/` merge into **`docs/`** per [2026-05-06-monorepo-design.md](../specs/2026-05-06-monorepo-design.md). **`nomlog-app/modules/nomlog-health`** stays a **`file:`** dependency of the app (no separate workspace entry required unless you choose to promote it later).

**Tech Stack:** pnpm 9.x, Turborepo 2.x, Node >= 20.19 (align with `nomlog-web` `engines`), existing Expo 53 / Next 16 / Express API unchanged at runtime.

**Execution order note:** The design spec lists “workspace shell” before “unify doc paths.” For fewer edits to `pnpm-workspace.yaml`, you may **unify `docs/` immediately after fixing Git (Task 2–3)** so the workspace file names `docs` once. The tasks below follow that **recommended** order.

**Rollback:** Keep work on a long-lived branch until root `pnpm install`, `turbo run lint`, app/API/web smoke checks pass. Do not delete `yarn.lock` files until `pnpm-lock.yaml` is committed and validated. Render/EAS dashboards: revert build commands to prior `yarn` invocations if needed.

**Where this file lives:** Before **Task 3**, it is under `docs/superpowers/plans/`; Task 3 moves it to **`docs/plans/`** with the other plans. The link to the design spec uses `../specs/` (same parent folder as `plans/`).

### Task 4: Root workspace shell (pnpm + Turbo)

**Files:**
- Create: `package.json`, `pnpm-workspace.yaml`, `turbo.json`, `packages/.gitkeep`

- [ ] **Step 1:** Create `package.json` at repo root:

```json
{
  "name": "nomlog",
  "private": true,
  "packageManager": "pnpm@9.15.9",
  "engines": {
    "node": ">=20.19.0"
  },
  "scripts": {
    "build": "turbo run build",
    "lint": "turbo run lint",
    "typecheck": "turbo run typecheck"
  },
  "devDependencies": {
    "turbo": "^2.4.4"
  }
}
```

- [ ] **Step 2:** Create `pnpm-workspace.yaml`:

```yaml
packages:
  - "nomlog-app"
  - "nomlog-api"
  - "nomlog-web"
  - "docs"
  - "packages/*"
```

- [ ] **Step 3:** Create `turbo.json`:

```json
{
  "$schema": "https://turbo.build/schema.json",
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", ".next/**", "build/**"]
    },
    "lint": {
      "dependsOn": ["^build"]
    },
    "typecheck": {
      "dependsOn": ["^build"]
    }
  }
}
```

**Note:** If `lint` / `typecheck` should not depend on `^build` for your workflow, drop `dependsOn` on those tasks after measuring—some teams use `dependsOn: []` for lint only.

- [ ] **Step 4:** Create `packages/.gitkeep` (empty file).

- [ ] **Step 5:** Commit root config **before** first install (workspace lists `docs`, which needs `docs/package.json` from Task 5).

```bash
git add package.json pnpm-workspace.yaml turbo.json packages/.gitkeep
git commit -m "chore: add pnpm workspace and turbo root (pre-install)"
```
