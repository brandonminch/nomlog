# Product Roadmap

This directory holds product roadmap documentation for Nomlog: existing features, planned work, ideas to iterate on, and important decisions.

Project plans and architecture live in [../../Documents/](../../Documents/); this directory is the **living product roadmap** and single place for feature inventory, roadmap, and ADRs.

## Contents

| Doc | Purpose |
|-----|--------|
| [Existing features](existing-features.md) | App + API feature inventory (links to full list) |
| [Roadmap](roadmap.md) | Planned features and release priorities |
| [MVP](mvp.md) | Must-ship list (tracks what qualifies as MVP) |
| [Inbox](inbox.md) | Captured TODOs not yet stories/PRDs |
| [Ideas](ideas.md) | Ideas to iterate on, experiments, backlog |
| [Decisions](decisions/) | Important decisions (ADR-style), one file per decision |
| [User stories](../stories/README.md) | Implementation-ready user stories grouped by feature |

## How to use

- **Existing features** — See [existing-features.md](existing-features.md) for a short overview and link to the full [FEATURES.md](../FEATURES.md) in docs.
- **Roadmap** — Edit [roadmap.md](roadmap.md) to track what you’re building next and later.
- **MVP** — Maintain [mvp.md](mvp.md) as the canonical must-ship list; mark stories with `release: mvp`.
- **Inbox** — Add real TODOs to [inbox.md](inbox.md) when they’re not ready for stories/PRDs yet; triage and promote regularly.
- **Ideas** — Add to [ideas.md](ideas.md) for uncommitted ideas or “maybe someday” items.
- **Decisions** — Add a new numbered ADR in [decisions/](decisions/) when you make an important, lasting decision; see [decisions/README.md](decisions/README.md) for format.
